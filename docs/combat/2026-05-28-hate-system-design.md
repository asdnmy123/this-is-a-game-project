## 一、核心规则

### 1.1 什么是有仇必报

每个敌人对场上所有存活的我方单位维护一个**仇恨值**（非负整数）。轮到敌人行动时，它攻击仇恨值最高的目标。

仇恨系统替代站位系统——没有前排后排的概念，谁拉的仇恨多，敌人就打谁。

### 1.2 两条基本原则

1. **各算各的** — 每个敌人独立维护一份仇恨表。敌人 A 可能仇恨玄甲，敌人 B 可能仇恨素问，不互相影响。
2. **嘲讽覆盖仇恨** — 被嘲讽的敌人无视仇恨值，强制攻击玄甲。嘲讽结束后回到仇恨判断。

### 1.3 仇恨的起与落

- 仇恨在战斗中**实时产生**（造成伤害/治疗等行为时立即计算）
- 仇恨在**每轮结束阶段统一衰减**
- 仇恨不会清零，只会衰减和重新生成
- 战斗结束后仇恨清空

## 二、仇恨值生成

### 2.1 产生仇恨的行为

| 行为 | 仇恨值 | 生效范围 | 说明 |
|------|--------|---------|------|
| 造成伤害 | `damage × class_multiplier` | 仅被命中的敌人 | 打谁谁记仇 |
| 治疗队友 | `heal × 0.35`（向下取整） | 全体存活敌人 | 降低被动仇恨，防止素问群奶 OT |
| 施加护盾 | `shield × 0.3`（向下取整） | 全体存活敌人 | 护盾也有存在感 |
| 施加 Buff/Debuff | 固定 30 | 全体存活敌人 | 放技能就有微量仇恨 |
| 使用嘲讽技能 | 见下文 2.3 | 全体敌人 | 强制建立仇恨 |
| 防御（不行动） | 0 | — | 怂着不招恨 |

> 护盾量和治疗量一样算"对团队的贡献"，所以对所有敌人均生效。这和"打谁谁记仇"的伤害逻辑不同，注意区分。

### 2.2 职业仇恨倍率

造成伤害时的 `class_multiplier`：

| 职业 | 倍率 | 理由 |
|------|------|------|
| 玄甲 | **×3** | 玄甲本身伤害低（DEF 转换伤害），靠倍率补上仇恨，才能拉住怪 |
| 方士 | ×0.7 | AOE 多目标总仇恨仍然较高，降低单目标倍率防止被全员集火 |
| 惊鸿 | ×1 | 标准倍率，单体高爆发在单目标上可能 OT |
| 素问 | ×0.5 | 素问伤害可以忽略，此条主要防止普攻奶模式下的异常仇恨 |

**为什么玄甲 ×3 而不是别的值？**

验算（基于 Lv40 四星数值，均无装备/Buff）：

```
场景：玄甲 vs 惊鸿 抢仇恨

玄甲盾击（DEF×80%）打玄甲级目标：
  damage = 400 × 80 / 100 = 320
  Step 3 减免后 ≈ 320 × 500 / (400 + 500) = 177
  仇恨 = 177 × 3 = 531

惊鸿破甲箭（ATK×220%）打方士级目标（暴击）：
  damage = 440 × 220 / 100 = 968
  减免后 ≈ 968 × 500 / (200 + 500) = 691
  暴击后 ≈ 691 × 160 / 100 = 1106
  仇恨 = 1106 × 1 = 1106
```

玄甲一次普攻约 531 仇恨，惊鸿一次暴击约 1106 仇恨。爆发期 OT 是可能的，但玄甲有**嘲讽**这个王牌来拉回。如果不希望 OT 太容易发生，数值策划可以调整 ×3 为 ×4 或 ×3.5，在后续调优阶段决定。

### 2.3 嘲讽技能的仇恨影响

玄甲使用**铁壁嘲讽**时：

1. **核心效果**：全体敌人被嘲讽，持续期间强制攻击玄甲（详见第四章）
2. **仇恨效果**：将玄甲对所有敌人的仇恨值统一设为 **当前场上最高仇恨值 + 500**

这个设计保证了：
- 嘲讽结束后，玄甲的仇恨在全体敌人中仍然是领先的
- 但如果嘲讽期间玄甲完全不输出（只防御），而 DPS 持续输出，嘲讽结束后仍可能 OT
- 500 的硬加成确保嘲讽后有一个明确的仇恨缓冲区

### 2.4 极端数值：仇恨逃逸保护（安全阀）

为防止极端情况（如玄甲被控、DPS 连续暴击再暴击）导致仇恨完全失控，增加一条保护规则：

**其他玩家对某敌人的仇恨值，不超过玄甲对该敌人仇恨值的 3.0 倍。** 如果计算后超过此上限，强制压到上限值。

```
设玄甲对敌人 A 的仇恨为 2000
则其他玩家对敌人 A 的仇恨上限 = 2000 × 3.0 = 6000
```

**适用条件：**
- 此上限**仅当玄甲对该目标的仇恨值 > 0 时生效**。如果玄甲从未攻击过该目标（仇恨=0），则其他玩家对该目标**不设上限**，可自由建立仇恨。
- 如果队伍中**没有玄甲职业**，此上限不适用（仇恨自由竞争）。
- 如果队伍中有**多个玄甲**，上限按玄甲中对当前目标仇恨最高的那个计算。

- 此上限不限制嘲讽（嘲讽是强制判定，不走仇恨）
- 调整说明：从 2.5× 上调至 3.0×，给 DPS 在爆发期更多的 OT 空间，同时保留安全兜底
- 目的是防止极端数值下仇恨完全失控，降低调试难度，但不限制正常 OT 玩法

## 三、玄甲的职业优势：仇恨闭环

### 3.1 玄甲凭什么拉住怪

玄甲的仇恨优势来自三个层面：

```
           ┌─────────────────────────┐
           │    铁壁嘲讽（主动王牌）    │
           │  持续 2 轮 + 仇恨 +500   │
           └───────────┬─────────────┘
                       │
           ┌───────────▼─────────────┐
           │    伤害仇恨 ×3 倍率      │
           │  盾击/主动技都能拉仇恨    │
           └───────────┬─────────────┘
                       │
           ┌───────────▼─────────────┐
           │  百战之躯（越残越硬）     │
           │  被集火时受伤减免 +30%    │
           └─────────────────────────┘
```

1. **嘲讽** — 主动拉怪，2 轮硬控 + 仇恨值直接登顶
2. **×3 倍率** — 即使嘲讽 CD 期间，正常输出也能维持仇恨
3. **百战之躯** — 被集火时越打越硬，不会因拉怪而暴毙

这三个层次形成一个**仇恨闭环**：嘲讽建仇 → 嘲讽期间维持输出 → 嘲讽结束后仇恨不掉 → CD 转好再嘲讽。

### 3.2 技能吃 DEF 与仇恨的关系

玄甲用 DEF 替代 ATK 计算技能伤害，本身是为了"堆防不浪费"。这套机制与仇恨系统的配合：

- 玄甲堆 DEF → 技能伤害提高 → 仇恨自然提高
- **不需要引入"额外嘲讽值"之类的第二套属性**
- 装备选择简单直接：堆 DEF = 又能扛又能拉

这是刻意设计：不让玩家在"堆防御"和"拉仇恨"之间做两难选择。

## 四、敌人 AI 目标选择逻辑

### 4.1 标准 AI（所有普通敌人）

```
轮到敌人行动时：

1. 检查是否被嘲讽
   ├─ 是 → 目标 = 玄甲（跳过仇恨判断）
   │       如果玄甲已阵亡 → 嘲讽自动失效，走仇恨判断
   └─ 否 → 进入 2

2. 仇恨判断
   选择仇恨值最高的我方单位
   如果有多个仇恨并列最高 → 在这些目标中随机选一个

3. 行动选择
   普攻和单体技能 → 攻击选定目标
   AOE 技能 → 正常释放（不受目标选择影响，AOE 打全场）
```

**AI 行为伪代码：**

```python
def choose_target(enemy, player_units):
    # Step 1: 嘲讽检查
    if enemy.taunted:
        xuanjia = find_unit(player_units, "玄甲")
        if xuanjia and xuanjia.is_alive:
            return xuanjia
        # 玄甲已死，嘲讽失效，走正常仇恨
    
    # Step 2: 仇恨判断
    max_hate = max(u.hate_table[enemy.id] for u in player_units)
    candidates = [u for u in player_units if u.hate_table[enemy.id] == max_hate]
    return random.choice(candidates)
```

### 4.2 特殊 AI 逻辑（扩展预留）

某些敌人可以有特殊行为，但基础框架不变：

| 特殊行为 | 实现方式 | 例子 |
|---------|---------|------|
| 专打奶妈 | 素问的仇恨值在计算时 ×2（表面值不变，敌人选目标时翻倍看待） | 刺客型敌人 |
| 随机攻击 | 无视仇恨值，纯随机 | 狂暴/混乱状态 |
| 打最低血量 | 无视仇恨，选 HP 百分比最低的目标 | 斩杀型敌人 |
| 全体 AOE | 技能本身就是 AOE，不需要选目标 | Boss 技能 |

**这些特殊 AI 不改变基础仇恨系统**，只在敌人的行为配置中标记。后续设计敌人技能时具体定义。

## 五、仇恨衰减

### 5.1 衰减规则

**每轮结束阶段**，所有仇恨值统一乘以 80%（向下取整）：

```
new_hate = floor(old_hate × 80 / 100)
```

衰减倍率 80% 意味着：
- 每轮仇恨保留 80%，大约 3 轮后仇恨降到原来的一半左右（0.8³ = 0.512）
- 如果某个玩家完全不产生新的仇恨，3 轮后它的仇恨基本不再是威胁
- 主动产生仇恨的玩家（玄甲持续输出）会自然拉开差距

### 5.2 衰减的目的

1. **防止仇恨永久固化** — 如果一点仇恨永远存在，早期爆发高的角色可能从头到尾被追着打
2. **给玄甲自然累积优势** — 玄甲每轮稳定产仇恨（×3 倍率 + 嘲讽），没仇恨压力的角色（如控住后不打）的仇恨会被时间稀释
3. **让 OT 后的救场有意义** — 如果 DPS 爆发 OT 了，只要它停手 2-3 轮，玄甲就能自然追回

### 5.3 不减的情况

- 被嘲讽的敌人的仇恨值**正常衰减**（嘲讽是强制硬控，和仇恨值无关）
- 仇恨值为 0 的不变（floor(0 × 0.8) = 0）
- 仇恨不会衰减到负数

## 六、OT（Over-Threat / 仇恨超越）

### 6.1 什么情况会 OT

OT 发生条件：**某个非玄甲职业对某敌人的仇恨值超过了玄甲**。

这不是 Bug，是**预期行为**。OT 机制保证了：

1. **玄甲不是锁仇恨** — 它需要操作和装备来维持仇恨，不是挂机坦克
2. **DPS 有操作空间** — 爆发期可以故意 OT 快速击杀，但风险自负
3. **提供决策深度** — 玩家要考虑"现在全力输出会不会 OT"

### 6.2 OT 后的行为

敌人 OT 后会立刻转向新目标（下一行动回合生效，不会在当前动画中突然转火）：

- 敌人 A 对玄甲的仇恨是 1000，对惊鸿的仇恨是 1300
- 敌人 A 轮到行动时，目标 = 惊鸿
- 惊鸿被打后，玩家需要应对

### 6.3 怎么拉回来

玄甲拉回仇恨的途径：

| 方式 | 操作 | 效果 |
|------|------|------|
| **嘲讽** | 使用铁壁嘲讽 | 强制嘲讽 2 轮 + 仇恨直接 +500 登顶 |
| **补输出** | 正常攻击 OT 目标 | 每轮 ×3 倍率输出逐步追回 |
| **等待衰减** | 让 OT 的队友停手/防御 | 对方仇恨衰减，玄甲自然反超 |

**玩家的操作链条：**

```
DPS 爆发 → OT → 敌人转火 DPS → DPS 受伤/危险
→ 玄甲开嘲讽拉回 → DPS 安全输出
→ 嘲讽结束 → 仇恨回归正常循环
```

### 6.4 故意 OT 的战术价值

玩家可以利用 OT 做战术操作：

- **集火秒 Boss**：全员爆发无视仇恨，争取在减员前打死 Boss（有风险，但可行）
- **让坦克临时扛别的怪**：玄甲嘲讽把高威胁怪拉走，DPS 快速处理低威胁怪
- **卖队友战术**：让一个残血单位去送死，换取其他人安全输出两轮（出现在硬核玩家操作中）

## 七、特殊场景

### 7.1 Boss 战

| Boss 类型 | 嘲讽免疫 | 对仇恨的影响 |
|-----------|---------|------------|
| 普通 Boss | **部分免疫** | 嘲讽持续仅 1 轮（减半），但仇恨 +500 效果仍在 |
| 高阶 Boss（主线 5 章后） | **完全免疫** | 嘲讽无效，玄甲只能靠 ×3 倍率 + 正常输出来维持仇恨 |
| 世界 Boss / 活动 Boss | 完全免疫 | 同上 |

**为什么 Boss 要免疫或部分免疫嘲讽？**
- 如果嘲讽对 Boss 全效，Boss 战变成"坦克无脑嘲讽，DPS 无脑输出"，缺少操作感
- Boss 免疫嘲讽后，玄甲需要通过**装备和操作**（稳定输出、保持仇恨）来拉怪
- 这是给坦克玩家的操作空间和成就感来源

**Boss 免疫嘲讽时，DPS 更要控仇恨：**
- 惊鸿爆发前要确认玄甲的仇恨够高
- 方士 AOE 会让所有 Boss 附属目标都仇恨自己，需要玄甲群嘲/逐个拉回
- 素问的群奶会拉全场仇恨，过度治疗等于自杀

### 7.2 多目标战（3v3+）

| 场景 | 仇恨表现 |
|------|---------|
| **玄甲单拉 vs 多个敌人** | 嘲讽全体，所有敌人强制打玄甲。嘲讽结束后，每个敌人独立仇恨判断 |
| **玄甲逐个击破** | 玄甲攻击敌人 A → 对 A 仇恨高，敌人 B/C 仇恨低。B/C 可能去打仇恨高的方士/素问 |
| **方士 AOE** | 方士 AOE 打中所有敌人 → 对所有敌人产生仇恨 → 大概率 OT 方士本人 |

**多目标战的典型操作流程：**
```
第 1 轮：玄甲嘲讽全体 → 所有敌人打玄甲，全员安全
第 2 轮：嘲讽持续，玄甲集火敌人 A，方士 AOE
第 3 轮：嘲讽结束，方士 AOE 拉高了所有敌人的仇恨 → 敌人 B/C 转火方士
         此时玄甲需要：要么再嘲（CD 中），要么快速干掉敌人 A 后逐个建立仇恨
```

这个设计导致**多目标战不能无脑 AOE**——方士需要和玄甲配合，不然 AOE 完就被集火。

### 7.3 AOE 场景

方士 AOE（烈焰符 ATK×130% 全体）的仇恨分析：

```
假设打 3 个敌人，各受到 300 伤害：

每个敌人对 方士 的仇恨 += 210（×0.7 倍率）
方士对 3 个敌人的仇恨都是 210

同期玄甲单点敌人 A，伤害 177：
敌人 A 对 玄甲 的仇恨 += 531（177 × 3）
敌人 B、C 对 玄甲 的仇恨 = 0（没被打）
```

结果：方士对敌人 B、C 的仇恨（210）仍高于玄甲（0），敌人 B、C 的下一个目标就是方士。

**结论：AOE 产生的高额群体仇恨是方士的软肋。** 这就是为什么方士身板最脆（HP 2800，全职业最低）——高 AOE 伤害的代价就是高仇恨 + 低生存。玩家需要安排玄甲的保护或者素问的及时治疗。

### 7.4 玄甲阵亡

如果玄甲在战斗中阵亡：

1. 敌人身上的嘲讽效果**立即消失**（Boss 完全免疫，普通敌人嘲讽也随玄甲死亡消失）
2. 所有敌人的仇恨表不变
3. 下一个被攻击的队友 = 当前仇恨最高的玩家
4. 素问是第二仇恨制造者（治疗全场仇恨），很可能成为下一个目标

玄甲阵亡后，残局转变为**"谁能扛住谁就是坦克"**的局面。素问可以靠治疗仇恨吸引火力，同时自奶维持，但这不是长久之计。

### 7.5 召唤物

预留：如果后续加入召唤物（如召唤兽、分身），召唤物不产生仇恨，敌人不会主动攻击召唤物。除非召唤物本身造成了伤害（那就算伤害者的仇恨）。

## 八、与回合流程的衔接

### 8.1 仇恨结算时间点

仇恨在回合流程中的位置：

```
轮开始阶段
  ├─ 胜负判定
  ├─ 触发"每轮开始时"效果
  └─ 重置"每轮一次"标记

行动阶段
  ├─ 生成出手序列（速度排序）
  ├─ 每个单位的回合：
  │    ├─ 回合开始（触发"回合开始时"效果）
  │    ├─ 敌人行动：根据当前仇恨值选择目标 ← ★ 实时读取仇恨表
  │    │             玩家行动：执行技能 → 立即产生仇恨  ← ★ 实时写入仇恨表
  │    └─ 回合结束（触发"回合结束时"效果）
  └─ （注意：新产生的仇恨在当前回合立即生效，影响后续单位的决定）

轮结束阶段
  ├─ 触发"每轮结束时"效果
  ├─ 仇恨值统一 × 80% ← ★ 仇恨衰减
  ├─ Buff/Debuff 计时减 1
  ├─ 移除到期效果
  └─ 胜负判定
```

**重点：** 仇恨在单位行动时实时产生，不需要等某个阶段结算。也就是说——如果方士在序列中比素问先行动，方士放了 AOE 产生仇恨，轮到素问行动时该仇恨已经存在了。

### 8.2 嘲讽的持续轮数

玄甲的**铁壁嘲讽**（来自四大职业设计）：

- **持续 2 轮**：从使用回合开始生效，到使用回合的轮结束阶段减 1，持续到下一轮结束
- **冷却 4 轮**：从使用回合开始计算，4 轮后可用
- **覆盖节奏**：

```
第 1 轮：使用铁壁嘲讽 → 嘲讽生效（持续 2 轮）
          轮结束：嘲讽剩余轮数 -1 → 剩余 1 轮
第 2 轮：嘲讽仍然生效
          轮结束：嘲讽剩余轮数 -1 → 剩余 0 轮，嘲讽移除
第 3 轮：无嘲讽 → 纯仇恨阶段
第 4 轮：无嘲讽 → 纯仇恨阶段
第 5 轮：铁壁嘲讽冷却结束（可用）
```

所以玄甲的嘲讽节奏是：**2 轮嘲讽 + 2 轮空窗 = 4 轮循环**。在空窗期，玄甲靠 ×3 倍率维持仇恨，DPS 需要控制输出节奏。

### 8.3 玩家先手软保底对仇恨的影响

回合流程设计中有"玩家方至少有一人先于敌人行动"的软保底。这对仇恨的影响：

1. **第一轮玄甲可能先手嘲讽**：如果玄甲被保底提前，可以在大部分敌人行动前建立嘲讽，保护全队
2. **惊鸿先手的仇恨积累**：如果惊鸿被保底先手，它的爆发可能在没有坦克仇恨的情况下直接把怪拉到自己身上。**这是风险**——先手 DPS 可能先 OT。
3. **玩家需要根据阵容调整策略**：如果队伍里有惊鸿，玄甲最好在惊鸿之前或紧跟其后建立仇恨。

## 十、设计理由

### 10.1 为什么用仇恨值而不是站位系统

| 方面 | 站位系统 | 仇恨系统（本方案） |
|------|---------|-----------------|
| 实现成本 | 需要前后排逻辑、位置切换、AOE 范围判定 | 每人一个整数字段，纯数值计算 |
| 策略深度 | 坦克在前排扛，脆皮在后排 | 坦克需要主动操作维持仇恨，不是挂机扛 |
| 操作反馈 | 坦克玩家："我站好了，你们打" | 坦克玩家："我要输出了、要嘲讽了、要保人了" |
| 适配字符风格 | 极简字符下"站位"表现力有限 | 仇恨值不需要视觉表现，后台计算即可 |

### 10.2 为什么坦克仇恨没有做成"绝对锁定"

- 绝对锁仇恨（坦克仇恨无限大，DPS 永远 OT 不了）的问题：坦克变成挂件，DPS 无脑输出，战斗没有操作感
- OT 是回合制战斗的重要组成部分——它强迫玩家思考"什么时候全力输出，什么时候控制节奏"
- 让玩家 OT 后自救/被坦克救回来，这个过程本身就有成就感

### 10.3 为什么用比例衰减而不是固定值衰减

- 比例衰减（×80%）对大数值和小数值都有效：早期仇恨差异小，衰减不明显；后期仇恨差异大，衰减效果显著
- 固定值衰减（每轮减 200）在后期会不够用（总仇恨 5000 减 200 感觉不到），前期又太狠（总仇恨 500 减 200 直接半残）
- 80% 是个合理的初值，容易调优

### 10.4 为什么治疗和 Buff 产生"全体仇恨"

- 如果治疗只对被治疗者的敌人产生仇恨，多目标战计算复杂（需要追踪每个敌人是否"看到"了治疗）
- 全体仇恨实现简单：治疗发生时，对所有敌人 + 仇恨值
- 从世界观角度说得通：敌人看到奶妈在回血，当然想先打奶妈
- 这也给素问加了操作压力——不能无脑群奶，需要考虑仇恨
