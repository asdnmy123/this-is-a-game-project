# 伤害公式设计

日期：2026-05-28
状态：approved

---

## 一、涉及的核心属性

每个战斗单位参与伤害计算的基础属性：

| 属性 | 符号 | 含义 | 典型范围（1~50级） |
|------|------|------|-------------------|
| 攻击力 | ATK | 基础伤害来源 | 35 ~ 1500 |
| 防御力 | DEF | 减免受到的伤害 | 20 ~ 800 |
| 生命值 | HP | 归零阵亡 | 300 ~ 8000 |
| 暴击率 | CRIT | 触发暴击的概率（百分数, 如 5 = 5%） | 3% ~ 25% |
| 暴击伤害 | CRIT_DMG | 暴击时的额外伤害加成（百分数, 如 50 = +50%） | 50% ~ 80% |

> 速度（SPD）只决定出手顺序，不影响伤害，已在回合流程文档中定义。

## 二、防御常数（核心参数）

整个公式基于一个**防御常数** `DEF_CONST = 500`。

```
伤害减免率 = DEF / (DEF + DEF_CONST)
```

- DEF = 0 → 减免 0%，吃满伤害
- DEF = 250 → 减免 33%
- DEF = 500 → 减免 50%（收益递减开始明显）
- DEF → ∞ → 极限减免趋近 100%，但永远达不到

```text
DEF 0   | 0% 减免
DEF 100 | 16.7% 减免
DEF 250 | 33.3% 减免  
DEF 500 | 50% 减免
DEF 800 | 61.5% 减免
```

收益递减的设计意图：
- 堆防御的前期收益高，鼓励坦克堆防御
- 后期每点防御的边际收益降低，防止高等级后完全打不动
- 公式永远不产生 0 伤害或负伤害，不需要额外 clamp

## 三、完整伤害计算流水线

所有伤害计算走同一个流水线，每个步骤都只做**整数运算**（先乘后除，避免精度丢失）。

```
Step 0: 获取最终面板属性（含装备 + Buff 修正）
  final_ATK = base_ATK × (100 + ATK_percent_bonus) / 100 + ATK_flat_bonus
  final_DEF = base_DEF × (100 + DEF_percent_bonus) / 100 + DEF_flat_bonus
  （其余属性类推）

Step 1: 技能基础伤害
  base_damage = final_ATK × skill_multiplier / 100

Step 2: 防御穿透（仅方士生效）
  effective_DEF = final_DEF × (100 - PENETRATION) / 100
  非方士职业: effective_DEF = final_DEF

Step 3: 防御减免
  reduced_damage = base_damage × DEF_CONST / (effective_DEF + DEF_CONST)

Step 4: 等级补正
  level_factor = clamp(100 + (attacker_Lv - defender_Lv) × 2,  60, 140)
  reduced_damage = reduced_damage × level_factor / 100

Step 5: 暴击判定
  如果 roll(1,100) ≤ CRIT:
    reduced_damage = reduced_damage × (100 + CRIT_DMG) / 100

Step 6: 随机波动
  variance = random(95, 105)   // ±5%
  reduced_damage = reduced_damage × variance / 100

Step 7: Buff/Debuff 最终修正
  // 所有"造成的伤害+X%"和"受到的伤害+X%"在此叠加（加法）
  // 减伤效果（"受到的伤害-X%"）也走 debuff_dmg_taken_bonus，取负值
  total_bonus = sum(attacker_dmg_bonus) + sum(defender_dmg_taken_bonus)
  reduced_damage = reduced_damage × (100 + total_bonus) / 100

Step 8: 保底
  final_damage = max(1, reduced_damage)   // 最少打 1 点
  结果向下取整（丢弃小数）
```

**伪代码（JavaScript 友好）：**

```javascript
function calcDamage(attacker, defender, skillMult) {
  // Step 1
  let base = Math.floor(attacker.atk * skillMult / 100);

  // Step 2
  let effDef = defender.def;
  if (attacker.class === 'FANG_SHI') {
    effDef = Math.floor(defender.def * (100 - attacker.penetration) / 100);
  }

  // Step 3
  let dmg = Math.floor(base * DEF_CONST / (effDef + DEF_CONST));

  // Step 4
  let lvFactor = 100 + (attacker.level - defender.level) * 2;
  lvFactor = Math.max(60, Math.min(140, lvFactor));
  dmg = Math.floor(dmg * lvFactor / 100);

  // Step 5
  if (Math.random() * 100 < attacker.crit) {
    dmg = Math.floor(dmg * (100 + attacker.critDmg) / 100);
  }

  // Step 6
  let variance = 95 + Math.floor(Math.random() * 11); // 95~105
  dmg = Math.floor(dmg * variance / 100);

  // Step 7
  let bonus = attacker.dmgDealtBonus + defender.dmgTakenPenalty;
  dmg = Math.floor(dmg * (100 + bonus) / 100);

  // Step 8
  return Math.max(1, dmg);
}
```

## 四、职业差异化

### 4.1 四职业参数表

| 参数 | 玄甲（坦） | 方士（法） | 惊鸿（弓） | 素问（奶） |
|------|-----------|-----------|-----------|-----------|
| 默认暴击率 | 3% | 5% | 15% | 3% |
| 默认暴击伤害加成 | +50% | +50% | +60% | +50% |
| 防御穿透率 | 0% | **30%** | 0% | 0% |
| 技能倍率特点 | 部分技能吃 DEF | 高倍率 | 极高单体倍率 | 治疗倍率 |

### 4.2 各职业的伤害角色

**玄甲（坦克）：**
- 标准流水线，无穿透，无额外暴击
- 区别在于面板属性：高 DEF、高 HP、低 ATK
- 部分技能可能以 DEF 代替 ATK 计算（如盾击 → `damage = DEF × skill_mult / 100`），这是技能层面的设计，不改变通用公式

**方士（法师）：**
- 核心特色：**防御穿透 30%**（无视目标 30% 防御）
- Step 2 中 `effective_DEF = DEF × 70%`
- 低 DEF 身板意味着被攻击时掉血明显
- 技能倍率整体高于惊鸿（但 AOE 技能倍率较低）

**惊鸿（弓手）：**
- 核心特色：**高暴击**（15% 基础，其他人 3-5%）
- 暴击伤害加成比其他人高 10 个百分点
- 技能倍率单体最高（点杀特化），但无 AOE

**素问（奶妈）：**
- 核心特色：治疗（见第八章）
- 输出能力弱，伤害计算走标准流水线，无特殊加成
- 低 ATK + 低暴击 = 伤害可忽略

### 4.3 穿透效果验算

方士 30% 穿透的实际效果（DEF_CONST = 500）：

```
目标 DEF | 无穿透减伤率 | 有穿透(30%)后减伤率 | 穿透带来的伤害提升
---------|-------------|-------------------|-----------------
200      | 28.6%       | 21.9%             | 约 +9.4%
400      | 44.4%       | 35.9%             | 约 +15.4%
600      | 54.5%       | 45.7%             | 约 +19.4%
800      | 61.5%       | 52.8%             | 约 +22.7%
```

目标越硬，穿透收益越大。符合直觉——法师克坦克。

## 五、暴击机制

### 5.1 暴击率

每职业基础暴击率不同（见上表），装备/被动可以增加。

暴击判定：每次攻击独立判定，`roll(1, 100) ≤ CRIT` 时触发。

### 5.2 暴击伤害

基础暴击伤害加成 = +50%（即暴击时伤害 × 1.5）。
惊鸿额外 +10%，即 +60%（×1.6）。

暴击伤害加成可叠加（装备、Buff 等），加法叠加：
```
实际 CRIT_DMG = 50 + sum(所有暴击伤害加成)
总暴击倍率 = (100 + 实际 CRIT_DMG) / 100
```

### 5.3 暴击与穿透的叠加

方士的穿透和暴击可以同时生效：
- Step 2 先算穿透 → Step 5 再算暴击
- 穿透=1.3x 有效伤害 × 暴击=1.5x → 综合 ≈ 1.95x 对同一目标
- 但方士暴击率低（5%），爆发看脸
- 惊鸿暴击率 15%，穿透 0%，稳定高暴击但打高防目标衰减明显

## 六、等级补正

### 公式

```
level_factor = clamp(100 + (attacker_Lv - defender_Lv) × 2,  60, 140)
```

### 效果表

| 等级差 | level_factor | 效果 |
|--------|-------------|------|
| +20 级 | 140（上限） | 高等级打低级，伤害 +40% |
| +10 级 | 120 | 伤害 +20% |
| 同级 | 100 | 无修正 |
| -10 级 | 80 | 伤害 -20% |
| -20 级 | 60（下限） | 低级打高级，伤害 -40% |

### 设计意图

- 每差 1 级 = 2% 伤害修正，衰减平滑
- 上限/下限防止等级碾压或等级压制过于极端
- 同级对战无修正，方便平衡
- 玩家越级挑战 boss（高 5-10 级）会感到压力但并非不可能
- 玩家回头刷低等级图会明显轻松但不会一刀秒杀（维持爽感不破坏游戏性）

## 七、最终伤害修正（Buff/Debuff）

### 7.1 修正分类

Buff/Debuff 对伤害的影响分为三类：

**A. 面板属性类**（在 Step 0 作用）：
- 攻击力 ±X%（影响 ATK，进而影响所有技能伤害）
- 防御力 ±X%（影响生存）

**B. 增伤/减伤类**（在 Step 7 作用）：
- "造成的伤害 +X%" → 加在 `attacker_dmg_bonus`
- "受到的伤害 +X%"（易伤）→ 加在 `defender_dmg_taken_bonus`
- 同类加法叠加：`+15% 造成伤害 + 20% 造成伤害 = 35%`
- 异类之间也是加法叠加（为简化计算）

**C. 特殊类**（技能私有的修正）：
- "对首领目标伤害 +X%"
- "目标生命 > 50% 时伤害 +X%"
- 这些也在 Step 7 叠加到 `total_bonus` 中

### 7.2 治疗加成

- "治疗效果 +X%" 不参与伤害公式，单独作用于治疗结算

## 八、治疗公式

### 8.1 基础治疗

```
heal = caster_ATK × heal_multiplier / 100
```

- 和伤害共用 ATK，不引入额外属性
- 素问 ATK 偏低，所以基础治疗量有限，需要堆叠"治疗效果"来提升

### 8.2 与其他属性的关系

- 不受目标 DEF 影响（治疗不经过防御减免）
- 不受等级补正影响
- 不受暴击影响（默认不能暴击治疗，素问被动可解锁）
- 受 "治疗效果 +X%" 修正：`heal = heal × (100 + heal_bonus) / 100`
- **不受"受到的伤害+X%"类效果影响**（易伤/debuff 只影响伤害结算，不影响治疗）

### 8.3 治疗流水线

```
Step 1: base_heal = final_ATK × heal_multiplier / 100
Step 2: base_heal = base_heal × (100 + attacker_heal_bonus) / 100
Step 3: base_heal = max(1, floor(base_heal))    // 最少回 1
```

## 九、验算示例

以下验算均基于一个假设的 Lv40 属性表（四星角色，无装备、无 Buff）：

| 属性 | 玄甲 Lv40 | 方士 Lv40 | 惊鸿 Lv40 | 素问 Lv40 |
|------|----------|----------|----------|----------|
| ATK | 320 | 560 | 440 | 280 |
| DEF | 400 | 200 | 280 | 280 |
| HP | 4800 | 2800 | 3400 | 3600 |
| CRIT | 3% | 5% | 15% | 3% |
| CRIT_DMG | +50% | +50% | +60% | +50% |
| 穿透 | - | 30% | - | - |

### 示例1：方士打玄甲（法克坦，常规技能倍率 150%）

```
Step 1: base = 560 × 150 / 100 = 840
Step 2: eff_DEF = 400 × 70% = 280
Step 3: dmg   = 840 × 500 / (280 + 500) = 840 × 500 / 780 = 538
Step 4: 同级, factor = 100, dmg = 538
Step 5: 未暴击（95% 概率）
Step 6: var = 100（取中间值）, dmg = 538
Step 7: 无额外修正
Step 8: final = 538

结果：方士打玄甲 538 伤害（玄甲 HP 4800 → 4262，约 11.2%）
```

### 示例2：惊鸿打方士（弓点法师，技能倍率 200%）

```
Step 1: base = 440 × 200 / 100 = 880
Step 2: 无穿透, eff_DEF = 200
Step 3: dmg = 880 × 500 / (200 + 500) = 880 × 500 / 700 = 628
Step 4: 同级, dmg = 628
Step 5: 假设暴击（15% 概率, CRIT_DMG = 60）→ dmg = 628 × 160 / 100 = 1004
Step 6: var = 100, dmg = 1004
Step 7: 无修正
Step 8: final = 1004

结果：惊鸿暴击打方士 1004 伤害（方士 HP 2800 → 1796，约 35.9%）
未暴击则 628（HP 2800 → 2172，约 22.4%）
```

### 示例3：玄甲打玄甲（坦克刮痧，普攻 100%）

```
Step 1: base = 320 × 100 / 100 = 320
Step 2: 无穿透, eff_DEF = 400
Step 3: dmg = 320 × 500 / (400 + 500) = 320 × 500 / 900 = 177
Step 4: 同级, dmg = 177
Step 5: 大概率无暴击
Step 6: var = 100, dmg = 177
Step 7: 无修正
Step 8: final = 177

结果：玄甲普攻打玄甲 177 伤害（HP 4800 → 4623，约 3.7%）
```

### 示例4：素问治疗（治疗倍率 150%）

```
Step 1: heal = 280 × 150 / 100 = 420
Step 2: 无额外治疗加成
Step 3: final = 420

结果：素问治疗 420（玄甲 4800 → 补满不可能，但约 8.75% 回血）
```

### 示例5：10 级等级压制（Lv40 打 Lv30）

```
取示例1 方士打玄甲，但目标只有 Lv30（假设属性相应降低，只看补正部分）：

Step 4: level_factor = 100 + (40-30) × 2 = 120
         dmg = 538 × 120 / 100 = 645

结果：因等级压制多了 107 伤害（+20%）。同时 Lv30 的 DEF 通常也更低，实际差距更大。
```

## 十、参数速查表

| 参数 | 值 | 说明 |
|------|-----|------|
| `DEF_CONST` | 500 | 防御减免曲线控制常数 |
| `LEVEL_SCALE` | 2 | 每差 1 级 ±2% 伤害 |

| `DEFAULT_CRIT` | 5 | 默认暴击率（5%，各职业不同） |

## 十一、设计要点总结

1. **单 DEF 系统**（不分物防/法防），方士用穿透来体现职业差异，而不是引入第二套防御属性。节省实现成本。
2. **整数运算友好**，所有步骤先乘后除，floor 保底，适合微信小程序 JS 环境。
3. **永不零伤**，防御再高也至少被强制扣 1 血，不会出现完全打不动的情况。
4. **伤害范围可控**，验算中单体爆发最高约 1000（Lv40 阶段），坦克 HP 4800，约 4-5 次攻击倒地，节奏合理。
5. **等级补正 ±2%/级**，配合属性成长，10 级差的实际伤害差距约 2 倍。鼓励玩家提升练度，回头刷低等级图有碾压感。跨级挑战则需要靠策略和配装弥补。
6. **暴击与穿透不冲突**，方士有穿透+低暴击（稳定的伤害提升），惊鸿无穿透+高暴击（不稳定的爆发），两者战术定位自然区分。
7. **后续扩展预留**：属性克制、伤害类型（物理/法术）、DoT 伤害等可以在此基础上新增模块，不破坏现有公式。
