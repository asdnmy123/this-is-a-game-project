## 一、设计总则

### 1.1 技能结构

每个角色固定拥有 4 个技能，按类型可分为三类：

| 类型 | 数量 | 释放方式 | 冷却 |
|------|------|---------|------|
| 普攻 | 1 | 每回合自动使用，玩家可选目标 | 无 |
| 主动技 | 2 | 玩家点击释放（或自动战斗 AI 自动释放） | 2~4 轮 |
| 被动技 | 1 | 条件触发，无需操作 | 无 |

合计：3 角色上场 = 6 个主动按键 + 3 个普攻 + 3 个被动。操作门槛低，手机屏幕够放。

### 1.2 已定稿的数值基础

以下数值直接从已定稿文档引用，本设计不再重复推导：

- **伤害公式**：`damage = ATK × mult / 100` → Step 2~8（公式详见伤害公式文档）
- **治疗公式**：`heal = ATK × mult / 100` → 经治疗加成修正（详见治疗基准文档）
- **仇恨公式**：伤害仇恨 = `damage × class_multiplier`，治疗仇恨 = `heal × 0.35`（详见仇恨系统文档）
- **职业仇恨倍率**：玄甲 ×3、方士 ×0.7、惊鸿 ×1、素问 ×0.5
- **素问治疗倍率**（已按治疗基准更新）：金针 100%、回春术 320%、益气汤 180%、悬壶 40%
- **属性参考**：均为 Lv40 四星角色无装备面板

### 1.3 属性速查（Lv40 四星）

| 属性 | 玄甲 | 方士 | 惊鸿 | 素问 |
|------|------|------|------|------|

| DEF | 400 | 200 | 280 | 280 |

| CRIT | 3% | 5% | 15% | 3% |

## 三、玄甲（坦克）技能详细设计

### 3.1 盾击 — 普攻

| 项目 | 内容 |
|------|------|
| 类型 | 普攻 |
| 目标 | 单体敌方 |
| 效果 | `base_damage = final_DEF × 80 / 100`，后续走 Step 2~8 标准流程 |
| 冷却 | 无（每回合可用） |
| 仇恨 | 伤害值 × 3（对被攻击目标） |
| 暴击 | 走标准暴击判定（玄甲基础 3%） |

**设计意图：**
玄甲 ATK 全职业最低（320），普攻如果用 ATK 就是刮痧，更关键的是拉不住仇恨。用 DEF 替代 ATK 后，盾击 Lv40 约打 177 伤害，产生 531 仇恨——够用且自然。玄甲只需要堆 DEF 就能兼顾扛打和拉怪，属性利用率不浪费。

**实现注意：**
- 盾击在 Step 1 用 DEF 代替 ATK，后续步骤完全复用伤害公式
- 不引入新逻辑，只是把 `final_ATK` 替换为 `final_DEF` 送入 `calcDamage()` 入口
- 字符串/UI 中盾击的伤害类型标注为"物理"，但不参与属性克制（本作无属性克制）

---

### 3.2 铁壁嘲讽 — 主动技

| 项目 | 内容 |
|------|------|
| 类型 | 主动（保护类） |
| 目标 | 全体敌方 |
| 效果 | **嘲讽效果**：全体敌人强制以玄甲为攻击目标（持续 2 轮）|
| | **增益效果**：自身 DEF +20%（持续 2 轮）|
| | **仇恨效果**：将玄甲对所有敌人的仇恨值设为当前场上最高仇恨值 + 500 |
| 冷却 | 4 轮 |
| 仇恨 | 嘲讽效果 + 自身 buff 30 仇恨（对全体敌人） |

**Lv40 约算（无装备）：**
- 玄甲 DEF = 400 → 铁壁嘲讽后 DEF = 480（持续 2 轮）
- 嘲讽期间敌人强制攻击玄甲（AOE 除外）
- 仇恨值被设为当前最高仇恨 + 500，保证嘲讽结束后玄甲仍在仇恨榜首

**设计意图：**
铁壁嘲讽是玄甲的核心技能，制造"安全窗口"——嘲讽持续期间，所有单体攻击都被玄甲吸收，队友安心输出。2 轮嘲讽 + 2 轮空窗 = 4 轮循环，空窗期靠 ×3 倍率维持仇恨。

双重效果（嘲讽 + 自增 DEF）让玄甲在嘲讽期间更硬，防止被集火秒掉。

**实现注意：**
- **嘲讽状态**：需要给每个敌人一个 `taunted` 标记和 `taunt_remaining` 计数器。轮到敌人行动时，先检查是否被嘲讽
- **嘲讽与 Boss**：普通 Boss 嘲讽减半（持续 1 轮），高阶 Boss 免疫嘲讽（仇恨 +500 效果仍在）
- **DEF+20% 自增益**：是一个独立 buff（ID: `BUFF_XJ_DEF_UP`），在 Step 0 影响面板属性。可与装备百分比加成叠加
- **仇恨值设置操作**：不是"增加"500，而是"设置为"max+500。这意味着如果嘲讽时玄甲仇恨已经最高，设置后他更高；如果别人仇恨更高，嘲讽后玄甲直接反超
- 如果玄甲在嘲讽期间阵亡，所有敌人的嘲讽标记立即清除

### 3.3 护盾援护 — 主动技

| 项目 | 内容 |
|------|------|
| 类型 | 主动（保护类） |
| 目标 | 单体友方（不含自身） |
| 效果 | 给目标施加护盾，吸收量 = `final_DEF × 150 / 100`，持续 2 轮 |
| 冷却 | 3 轮 |
| 仇恨 | 护盾吸收量 × 0.3（对全体敌人）+ 施加增益 30（对全体敌人） |

**Lv40 约算（无装备）：**
- 护盾吸收量 = 400 × 150% = 600
- 这个 600 护盾大约能吸收一次普攻伤害（同级敌人普攻约 200~400 伤害）

**设计意图：**
单体保人技能。当脆皮队友（方士、惊鸿）被集火或残血时，套盾吸收伤害，给素问争取治疗时间。冷却 3 轮，比铁壁嘲讽短，可更频繁使用。

之所以不给玄甲自己套盾——玄甲自己有嘲讽 + 百战之躯，不需要额外保护。给队友套盾体现的是"坦克保护别人"的定位。

**实现注意：**
- 护盾机制详见第七章
- 目标不能是自身（策划配置时限死）
- 如果目标已有护盾，新护盾替换旧护盾（不叠加）
- 护盾吸收量受玄甲 DEF 影响，所以玄甲堆 DEF 同时提升盾量和自身硬度，属性效率高
- 护盾值显示在角色血条上（UI 层用不同颜色），需要玩家清晰感知"我还有盾"

### 3.4 百战之躯 — 被动技

| 项目 | 内容 |
|------|------|
| 类型 | 被动 |
| 触发条件 | 受到伤害时（在伤害结算完成后触发） |
| 效果 | 每损失 10% 最大 HP，获得 +3% 受伤减免。上限 +30%（损失 100% HP 时达到）|
| | 减伤计算基于**受到本次伤害前的 HP 百分比** |
| 冷却 | 无（常驻效果） |
| 仇恨 | 无 |

**减伤效果计算示例：**

```
玄甲 HP 4800，当前 3600（75%），损失 25%

损失百分比：floor((4800 - 3600) / 480) = floor(2.5) = 2 → 2 个 10% 段
减伤：2 × 3% = 6%

下一个敌人攻击打上来，最终伤害再 × (100% - 6%)

如果当前 HP 降到 2400（50%），损失 50%：
减伤：5 × 3% = 15%
```

**设计意图：**
百战之躯是玄甲的"血条越少越硬"机制，防止坦克在残血时被一刀带走。这个机制与嘲讽体系配合：

- 嘲讽期间玄甲承受所有单体攻击 → 血线下降 → 百战之躯触发减伤 → 血线下降速度变慢
- 形成负反馈：越挨打越硬，不会出现"满血被秒杀"的极端情况
- 给素问留出治疗窗口——玄甲残血但减伤高，素问有时间奶回来

之所以"基于受到伤害前的 HP%"而不是"基于受到伤害后的 HP%"，是为了让减伤效果在本轮伤害中即时生效。如果基于受伤后计算，每次受伤都是一次"裸接伤害"，减伤只影响后续攻击，那第一下大伤害就直接要命了。

**五星强化预期：**
- 四星：+3% 每 10%（上限 +30%）
- 五星：+4% 每 10%（上限 +40%） + 每轮回复 3% HP（额外的恢复效果）

**实现注意：**
- 在 Step 7（Buff/Debuff 最终修正）中叠加到减伤效果中，以负值的 `defender_dmg_taken_bonus` 形式
- 需要战斗管理器在每次伤害前计算当前减伤百分比：`减伤 = floor((1 - 当前HP比例) × 10) × 3%`
- 如果回复了 HP，减伤也随之减少——不要缓存，每次受伤前实时计算
- HP 比例计算用整数运算：`floor((maxHP - currentHP) × 10 / maxHP)` 得到 "10% 段数"

## 四、方士（法师）技能详细设计

### 4.1 符箭 — 普攻

| 项目 | 内容 |
|------|------|
| 类型 | 普攻 |
| 目标 | 单体敌方 |
| 效果 | `base_damage = final_ATK × 100 / 100`，走标准伤害流程 |
| 冷却 | 无 |
| 仇恨 | 伤害值 × 0.7（对被攻击目标） |

**Lv40 约算（无装备）：**
- 打玄甲级目标（DEF 400）：ATK×100% = 500 → 经穿透减免后 ≈ 320 伤害 → 224 仇恨
- 打方士级目标（DEF 200）：ATK×100% = 500 → 经穿透减免后 ≈ 390 伤害 → 273 仇恨

**设计意图：**
标准的法系普攻，不出彩但够用。方士 ATK 全职业最高，即使普攻也有输出。仇恨倍率 ×0.7 是方士的生存保障——AOE 本来就会拉群体仇恨，单体普攻的仇恨再低一点，防止被单个敌人追着打。

**实现注意：**
- 走标准伤害流程，方士的 30% 穿透自动生效（Step 2）
- 无特殊逻辑

---

### 4.2 烈焰符 — 主动技

| 项目 | 内容 |
|------|------|
| 类型 | 主动（输出类） |
| 目标 | 全体敌方 |
| 效果 | `base_damage = final_ATK × 130 / 100`，对每个敌人独立走标准伤害流程 |
| 冷却 | 3 轮 |
| 仇恨 | 每个被命中的敌人分别产生 `damage × 0.7` 的仇恨 |

**Lv40 约算（无装备）：**
- 打 3 个不同目标：总伤害 ≈ 416+507+467 = 1390（取决于各目标 DEF）
- 总仇恨 ≈ 291+355+327 = 973（分散到 3 个敌人）

**设计意图：**
方士的招牌技能，全游戏唯一的伤害 AOE（惊鸿和玄甲无 AOE，素问只有群奶无伤害 AOE）。作用是压低全体敌方血线，为惊鸿的点杀创造机会。

冷却 3 轮的设定保证一场常规战斗能用 2~3 次，配合通灵被动可能刷到更多次。130% 倍率在单目标上低于普攻，但多目标时总收益极高。

**实现注意：**
- AOE 需要对每个目标独立计算伤害（各目标 DEF 不同，暴击分别判定）
- 每个目标产生独立的仇恨值
- 如果 AOE 杀死了多个敌人，通灵被动会触发多次（每次 -1 冷却）
- 性能注意：AOE 最多 6 个敌方目标，每个目标独立走一遍伤害流程，计算量可控
- 烈焰符是"全体 AOE"——对敌方所有存活单位生效，不需要指定目标

---

### 4.3 缚灵咒 — 主动技

| 项目 | 内容 |
|------|------|
| 类型 | 主动（输出 + 控制类） |
| 目标 | 单体敌方 |
| 效果 | `base_damage = final_ATK × 180 / 100`，走标准伤害流程 |
| | 额外：目标 SPD -20%，持续 2 轮 |
| 冷却 | 3 轮 |
| 仇恨 | 伤害值 × 0.7（对被攻击目标）+ 施加减益 30（对全体敌人） |

**Lv40 约算（无装备）：**
- 打方士级目标（DEF 200）：ATK×180% = 900 → 经穿透减免 → ≈ 702 伤害 → 491+30 = 521 仇恨
- 附加效果：目标 SPD 降低 20%（例如 SP150 → 120，掉到方士同档）

**设计意图：**
方士的控场技能。180% 单体倍率高于烈焰符（130%），但烈焰符是 AOE，缚灵咒是单体 + 控场。选择逻辑：需要减速控高速敌人时用缚灵咒，需要压全体血线时用烈焰符。

SPD -20% 的效果很显著——把一个高速敌人拉到中速，可以让己方在出手顺序上占据优势。对 Boss 同样生效（Boss 只是免疫嘲讽，不免疫减速 debuff）。

组合技：缚灵咒减速 + 惊鸿高速先手 = 先手优势翻倍。

**实现注意：**
- SPD -20% 是面板属性类的 debuff（Debuff ID: `DEBUFF_SPD_DOWN`），影响 Step 0 的速度计算，进而影响出手顺序
- 减速 debuff 不与同类型叠加（不叠加为 SPD -40%）
- 减速对出手顺序的影响实时生效——如果减速施放在行动阶段中途，不会改变当前轮的出手顺序（已排序完了），但会影响下一轮的排序
- 是否可净化：可以（素问的回春术可清除）

---

### 4.4 通灵 — 被动技

| 项目 | 内容 |
|------|------|
| 类型 | 被动 |
| 触发条件 | 方士本人击败任意敌方单位时 |
| 效果 | 方士所有主动技能（烈焰符、缚灵咒）的当前冷却 -1 轮 |
| | 同一轮内多个击杀可多次触发 |
| 冷却 | 无（被动） |
| 仇恨 | 无 |

**设计意图：**
方士的收割补偿机制。方士本身伤害不低（全职业最高 ATK），但暴击率低，收割能力不如惊鸿。通灵的设计鼓励方士在 AOE 压血线后补刀残血：

- 击杀一个敌人 → 烈焰符冷却从 3→2（缩短 1 轮）
- 击杀两个敌人 → 烈焰符冷却 3→1（甚至可以无视冷却）
- 连续击杀 → 刷出更多 AOE → 更多击杀 → 雪球效应

但这个雪球有天然限制——方士需要队友先压血线（自己是 AOE 不是点杀），而且收割需要恰好补到尾刀，不是每次都能触发。

**特别注意：通灵与追猎的互动：**
- 惊鸿击杀敌人 → 追猎触发追击 → 如果追击恰好把残血敌人打死 → 这是惊鸿的击杀，不是方士的击杀，所以通灵不触发
- 方士击杀敌人 → 通灵触发冷却缩减 → 追猎不触发（追猎是惊鸿的被动）
- 两个被动各自独立运作

**实现注意：**
- 需要挂钩到"击杀事件"——战门管理器在每次伤害结算后检查目标是否阵亡
- "方士本人击败"意味着不是友方助攻，是方士自己的技能造成最后一击
- 通灵减少的是"当前剩余冷却"，不是"总冷却"。例如烈焰符还剩 2 轮冷却，击杀后变为 1 轮
- 如果多个主动技能的冷却不同，各自独立减 1
- **不减少普攻的冷却**（普攻无冷却）
- 同一轮内多次击杀 → 多次触发，无上限

## 五、惊鸿（弓手）技能详细设计

### 5.1 速射 — 普攻

| 项目 | 内容 |
|------|------|
| 类型 | 普攻 |
| 目标 | 单体敌方 |
| 效果 | `base_damage = final_ATK × 100 / 100`，走标准伤害流程 |
| | 额外：本技能暴击率 +5%（实际暴击率 = 角色暴击率 + 5%） |
| 冷却 | 无 |
| 仇恨 | 伤害值 × 1（对被攻击目标） |

**Lv40 约算（无装备）：**
- 打方士级目标（DEF 200）：ATK×100% = 440 → 经减免 ≈ 314 → 314 仇恨
- 暴击率：15%（基础）+ 5%（速射加成）= 20%
- 如果暴击：314 × 160% = 502 伤害

**设计意图：**
一个自带 5% 额外暴击率的普攻，让惊鸿的普攻也有看脸爆发的快感。20% 的暴击率意味着大约每 5 次普攻就有一次暴击，手感明显。

与其他普攻的对比：
- 玄甲盾击：DEF×80%，主要用途是拉仇恨，伤害是副产品
- 方士符箭：ATK×100%，朴实无华的标准普攻
- 素问金针：治疗，不是伤害
- 惊鸿速射：ATK×100% + 5% 额外暴击，有机会出暴击惊喜

**实现注意：**
- 暴击率加成仅影响本技能，不影响其他技能或普攻全局
- 实现方式：进入伤害流程 Step 5 时，临时将角色暴击率 +5%
- UI 提示：技能图标或描述中标注"+5% 暴击率"

### 5.2 破甲箭 — 主动技

| 项目 | 内容 |
|------|------|
| 类型 | 主动（输出类） |
| 目标 | 单体敌方 |
| 效果 | `base_damage = final_ATK × 220 / 100`，走标准伤害流程 |
| 冷却 | 3 轮 |
| 仇恨 | 伤害值 × 1（对被攻击目标） |

**Lv40 约算（无装备）：**
- 打方士级目标（DEF 200）：ATK×220% = 968 → 经减免 ≈ 691 → 691 仇恨
- 暴击（15% 概率）：691 × 160% = 1106 伤害
- 约打方士 2800 HP 的 39.5%——接近半血
- 如果配上鹰眼（ATK+30%, CRIT+20%）：ATK = 440×1.3 = 572
  - ATK×220% = 1258 → 减免 ≈ 898 → 暴击率 35%
  - 暴击时：898 × 160% = 1437 伤害 → 打方士 2800 HP 的 51.3%

**设计意图：**
惊鸿的核心单体爆发技能。220% 的倍率是全游戏最高单体倍率（方士缚灵咒 180% 且带控，惊鸿纯粹追求伤害）。在有鹰眼 buff 的 combo 下，一箭打残脆皮是预期表现。

CD 3 轮：比鹰眼（4 轮）短，所以不是每次破甲箭都能配合鹰眼。玩家需要选择"什么时候开鹰然后爆发，什么时候直接打"。

**实现注意：**
- 标准单体伤害流程，无特殊逻辑
- 高倍率 + 高暴击 → 伤害方差大——这是刻意设计（弓箭手的职业幻想），但在自动战斗中 AI 无法评估方差，需要做好"不暴击时伤害不足"的心理预期

### 5.3 鹰眼 — 主动技

| 项目 | 内容 |
|------|------|
| 类型 | 主动（增益类） |
| 目标 | 自身 |
| 效果 | 自身 ATK +30%，暴击率 +20%，持续 2 轮 |
| 冷却 | 4 轮 |
| 仇恨 | 施加自身增益 30（对全体敌人） |

**设计意图：**
惊鸿的爆发前置技能。开启后大幅提升攻击和暴击率，为下轮（或本轮后续行动，如果速度足够快可以再次行动）的破甲箭做准备。

冷却 4 轮 + 持续 2 轮 → 2 轮空窗期。这意味着每 4 轮循环中有 2 轮是"爆发期"，2 轮是"平缓期"。玩家需要考虑：

- 开场直接开鹰眼 → 第一轮爆发延迟了一轮（浪费了先手优势）
- 先打一轮普攻再开鹰眼 → 第二轮爆发
- 什么时候开鹰眼最划算——取决于当前战场局势

组合技搭配：鹰眼 → 下轮破甲箭（暴击率提升到 35%，ATK+30%），预期伤害比无 buff 的破甲箭高约 40-50%。

**Buff 细节：**
- Buff ID: `BUFF_JH_HAWK_EYE`
- 两个效果（ATK+30%、CRIT+20%）绑定为同一个 buff，同时施加同时消失
- 持续 2 轮，可被净化吗？→ 建议**不可净化**（这是一个自增益，不是负面状态，不在素问回春术的净化范围内）

**实现注意：**
- ATK+30% 影响面板属性（Step 0），进而影响所有技能的伤害
- CRIT+20% 影响暴击判定（Step 5）
- 两个效果合并在一个 buff 里管理，减少 buff 槽位占用
- 如果惊鸿在鹰眼持续期间再次使用鹰眼（虽然 CD 4 轮很难做到），效果不叠加，重置持续时间为 2 轮
- UI 上需要清晰显示鹰眼剩余轮数和当前状态（图标亮显）

### 5.4 追猎 — 被动技

| 项目 | 内容 |
|------|------|
| 类型 | 被动 |
| 触发条件 | 惊鸿本人击败任意敌方单位时 |
| 效果 | 立即对另一个随机敌方存活单位发动一次普攻，伤害 = 正常普攻的 60% |
| | 速度 = 与普攻一致（不重新排序），继承被动暴击率加成（+5%） |
| | 每轮最多触发一次，不可连锁触发（追击击杀不触发第二次追击） |
| 冷却 | 每轮一次 |
| 仇恨 | 追击伤害值 × 1（对被攻击目标） |

**设计意图：**
惊鸿的收割被动——每轮第一个击杀触发的追击，可以看作"击杀后顺手补一刀"。60% 普攻伤害的追击已经很可观（约 188 伤害对中甲目标），用来补残血绰绰有余。

限制条件：
- **每轮最多一次**：防止惊鸿滚雪球太离谱（击杀→追击→再击杀→再追击……）
- **不可连锁**：追击本身造成的击杀不触发第二次追猎（防止无限连）
- **随机目标**：追击是"惊鸿视力范围内的另一个目标"，随机性让结果不那么可控

"随机"的设计取舍：如果是玩家手动选择目标，追击的自由度太高，每轮都能精准补刀残血。随机保留了一些不确定性，也防止玩家在自动战斗中微操追击。

**实现注意：**
- 需要"每轮一次"标记：每轮开始时重置（在轮开始阶段）
- 追击的伤害计算：取惊鸿的普攻流程，最后 × 60%（建议在 Step 1 就降低倍率，方便复用流程）
  - `base_damage = final_ATK × 60 / 100` → 走完整 Step 2~8
  - 相当于一个临时技能，倍率 60%，继承角色属性
- 随机目标选择：所有存活敌方单位（排除已被击杀的那一个），等概率随机
- 追击不占用行动次数——它是在击杀事件后立即插入的额外行动，不影响速度排序中的位置
- 如果追击打死敌人，通灵（方士被动）不触发，因为击杀者是惊鸿（通灵需要方士本人击杀）

## 六、素问（奶妈）技能详细设计

### 6.1 金针 — 普攻

| 项目 | 内容 |
|------|------|
| 类型 | 普攻（治疗类） |
| 目标 | 当前 HP% 最低的友方单位（自动选择，不可手动改目标） |
| 效果 | `heal = final_ATK × 100 / 100`，走治疗流程 |
| 冷却 | 无 |
| 仇恨 | 治疗量 × 0.35（对全体敌人） |

**Lv40 约算（无装备）：**
- 治疗量：280 × 100% = 280 → 经 +30% 治疗装备 = 364
- 对玄甲（4800 HP）：7.6% | 对方士（2800 HP）：13%
- 仇恨：280 × 0.35 = 98（对所有敌人）

**设计意图：**
让素问每回合都有事做——普攻是奶不是打人，符合素问"不输出"的定位。自动选择 HP% 最低的队友，保证治疗资源不浪费。

金针的目标选择策略："当前 HP% 最低"——比"当前 HP 值最低"更合理，因为坦克的 HP 上限高，残血看起来数值大但比例不一定最低。

**实现注意：**
- 目标选择：遍历所有友方存活单位，计算 `currentHP / maxHP`，选比值最小的
- 如果多个目标 HP% 相等（且都是最低），选先出手/位置靠前的（固定规则，减少随机）
- 治疗流程：走标准治疗公式 `heal = final_ATK × mult / 100` → 治疗加成修正

### 6.2 回春术 — 主动技

| 项目 | 内容 |
|------|------|
| 类型 | 主动（治疗类） |
| 目标 | 单体友方 |
| 效果 | `heal = final_ATK × 320 / 100`，走治疗流程 |
| | 额外：净化目标身上的 1 个减益效果（`Debuff` 标记） |
| 冷却 | 3 轮 |
| 仇恨 | 治疗量 × 0.35（对全体敌人）+ 净化效果 30（对全体敌人） |

**Lv40 约算（无装备）：**
- 治疗量：280 × 320% = 896
- 中等装备（+30% 治疗）：896 × 1.3 = 1165
- 对玄甲（4800 HP）：24.3% | 对方士（2800 HP）：41.6%
- 仇恨：896 × 0.35 + 30 = 343（对所有敌人）

**设计意图：**
素问的核心急救技能。320% 的高倍率确保一次治疗能把残血队友抬回安全线，附带的净化效果可以清除 debuff（如方士的减速、后续 Boss 的减益技能）。

CD 3 轮意味着回春术是"关键时刻用"的技能，不能每回合都放。玩家需要在"现在救人"和"留到更危险的时候"之间做选择——这是素问操作的核心策略点。

**净化规则：**
- 净化优先顺序：当目标身上有多个 debuff 时，按以下优先级净化：
  1. 控制类 debuff（如果能动，先解控）
  2. 减伤/dot 类的生存威胁 debuff
  3. 其他属性减益 debuff
- 己方增益效果不受净化影响
- 净化只清除 1 个 debuff（不是全部）

**实现注意：**
- 治疗 + 净化是两个效果先后执行：先治疗，后净化
- 净化的 debuff 选择逻辑（AI 自动模式）：按上述优先级选择
- 手动模式：玩家使用回春术时不需要选 debuff（自动选最高优先级的一个）
- 如果目标身上没有 debuff，回春术正常治疗（净化部分无效，但仇恨 30 仍产生）
- 回春术的净化不区分"可净化"和"不可净化"——所有`Debuff` 标记的效果都可净化。Buff 标记（正增益）不可净化

### 6.3 益气汤 — 主动技

| 项目 | 内容 |
|------|------|
| 类型 | 主动（治疗类） |
| 目标 | 全体友方 |
| 效果 | `heal = final_ATK × 180 / 100`，对每个队友独立走治疗流程 |
| 冷却 | 4 轮 |
| 仇恨 | 每个被治疗的队友分别产生 `heal × 0.35` 的仇恨（对全体敌人） |

**Lv40 约算（无装备）：**
- 全队治疗量：504/人 × 3 人 = 1512 总治疗量
- 中等装备（+30% 治疗）：786/人 × 3 人 = 2358 总治疗量
- 仇恨：504 × 0.35 × 3 = 529（对所有敌人，总计）

**设计意图：**
素问的群体治疗技能，应对敌方 AOE 场景。CD 4 轮说明这不是常驻技能——一场战斗通常用 1~2 次，在关键时刻抬全队血线。

180% 的群奶倍率在单目标上低于回春术（320%），但 3 目标的总治疗量（540%）远超回春术。代价是冷却更长（4 vs 3），仇恨更高（3 人份的治疗仇恨）。

注意：益气汤会产生大量群体仇恨（约 529），是素问 OT 的主要来源。如果连续使用益气汤，素问可能被敌人集火。这给玩家一个权衡：群奶保队友 vs 群奶拉仇恨。

**实现注意：**
- 对每个队友独立计算治疗量（各人的治疗加成可能不同）
- 仇恨是对全体敌人产生——即使某个队友满血被治疗（溢出），仇恨仍然产生
- 素问本人的治疗收益：益气汤也奶素问自己（如果素问是自己队友的目标）
- 如果素问自己残血 + 队友残血，益气汤可以一奶全体

### 6.4 悬壶 — 被动技

| 项目 | 内容 |
|------|------|
| 类型 | 被动 |
| 触发条件 | 素问行动结束时（每次轮到素问的回合并完成行动后触发） |
| 效果 | `heal = final_ATK × 40 / 100`，对所有友方生效 |
| 冷却 | 无（每回合素问行动结束时触发） |
| 仇恨 | 治疗量 × 0.35（对全体敌人） |

**Lv40 约算（无装备）：**
- 全队回血：112/人 × 3 人 = 336 总治疗量（每回合）
- 中等装备（+30% 治疗）：175/人 × 3 人 = 525 总治疗量
- 仇恨：112 × 0.35 × 3 = 117（对所有敌人，总计）

**设计意图：**
素问的被动续航。每回合自动回全队微量血，维持血线健康，防止队友被 AOE 慢慢蹭死。配合金针（普攻奶）形成"每回合至少 2 次治疗"的基础节奏：

```
素问每回合基础治疗量 = 金针(对单) + 悬壶(全队)
Lv40无装备: 280 + 336 = 616 总治疗/回合
```

悬壶的触发时机是"素问行动结束时"——意味着素问如果在本回合被击杀/控制，悬壶不会触发（因为行动未完成）。这增加了控/杀素问的战略价值。

悬壶的治疗量很小（40%），但胜在每回合都触发，持续收益。长期战斗的累积量不可忽视。

**五星强化预期：**
- 四星：40% ATK
- 五星：40% ATK + 额外效果（如"自身受治疗 +10%"等，具体在角色设计阶段定）

**实现注意：**
- 触发时机：素问完成自身行动（普攻或主动技能）后，立即触发悬壶
- 如果在素问行动前素问被击杀/控制，悬壶不触发（行动被跳过）
- 悬壶的仇恨是"被动产生"的——玩家不需要为悬壶的仇恨负责，它是被动的一部分
- 悬壶的治疗目标不包含已阵亡队友（战斗管理器中的"存活"判定）

## 七、护盾机制

护盾是一个独立的伤害吸收层，附着在角色 HP 之上。

### 7.1 核心规则

1. **吸收顺序**：护盾在角色实际 HP 之前吸收伤害。当角色受到伤害时：
   - 伤害先扣减护盾值
   - 护盾归零后，剩余伤害扣减角色 HP

2. **不叠加**：新护盾替换旧护盾（护盾值取新值，持续轮数重置），不存在"叠加护盾"

3. **不可治疗**：护盾不是 HP，不受治疗影响。护盾只能通过被伤害消耗或持续时间到期自然消失

4. **可被驱散**：护盾有独立的 buff ID（`BUFF_SHIELD`），可以被驱散类效果移除（待 Buff/Debuff 机制定稿时确认）

5. **护盾与减伤的关系**：
   - 伤害先经过伤害公式计算（含减伤效果）
   - 最终伤害再作用于护盾
   - 所以护盾 = 减伤生效后的额外吸收层

### 7.2 护盾生命周期

```
使用护盾技能 → 目标获得护盾 buff（ID: BUFF_SHIELD）
  ├─ 数值 = 施法者的 final_DEF × 150%
  └─ 持续 2 轮（轮末递减）

受到伤害时：
  护盾值 -= 最终伤害
  如果护盾值 ≤ 0 → 护盾消失，溢出伤害扣 HP

轮结束阶段：
  护盾持续轮数 -1
  如果持续轮数 ≤ 0 → 护盾消失
```

### 7.3 护盾的视觉表现

- 角色血条上方用额外颜色（建议浅蓝色）显示护盾值
- 护盾值不超过角色 HP 上限
- 护盾持续轮数用图标角标显示

### 7.4 边界情况

| 场景 | 处理方式 |
|------|---------|
| 盾击打护盾 | 护盾吸收伤害，玄甲打出"破盾"效果但无额外收益 |
| 护盾 + 百战之躯 | 护盾受伤不触发百战之躯（百战之躯基于 HP 损失，护盾掉了不算 HP 损失） |
| 护盾 + 治疗 | 护盾不影响治疗，直接回 HP |
| 护盾持续但不掉血 | 持续 2 轮后自动消失，护盾值浪费 |
| 新护盾覆盖旧护盾 | 旧护盾值被丢弃，取新值。不会出现"旧护盾 200 + 新护盾 600 = 800" |

## 八、技能升级系统

### 8.1 升级机制

每个角色有一个**技能等级**（Skill Level, 简称 SLv），范围 1~10。技能等级对角色所有 4 个技能生效（普攻 + 2 主动 + 1 被动同时升级）。

**角色等级 vs 技能等级：**
- 技能等级独立于角色等级（不自动跟随）
- 角色等级限制技能等级上限：`技能等级上限 = floor(角色等级 / 5)`（Lv50 时解锁 SLv10）
- 玩家需要消耗资源主动升级

### 8.2 每级提升内容

| 技能类型 | 每级提升 | 说明 |
|---------|---------|------|
| 普通攻击 | 倍率 +Y | 盾击 DEF×80% → DEF×(80+Y)%，依此类推 |
| 主动技能 | 倍率 +Z | 破甲箭 ATK×220% → ATK×(220+Z)% |
| 被动技能 | 效果值 +W | 百战之躯 +3%/10%HP → +(3+W)%/10%HP |

**具体每级提升值**（由数值策划在后续定稿精确值，此处只给大致方向）：

| 技能 | Lv1 倍率 | 每级提升 | Lv10 倍率 |
|------|---------|---------|-----------|
| 盾击 | DEF×80% | +4% | DEF×116% |
| 铁壁嘲讽 | DEF+20%（无倍率变化） | 持续轮数/效果不提升 | 不变（提升价值在 Z 值） |
| 护盾援护 | DEF×150% | +8% | DEF×222% |
| 方士符箭 | ATK×100% | +5% | ATK×145% |
| 烈焰符 | ATK×130% | +7% | ATK×193% |
| 缚灵咒 | ATK×180% | +8% | ATK×252% |
| 惊鸿速射 | ATK×100% | +5% | ATK×145% |
| 破甲箭 | ATK×220% | +8% | ATK×292% |
| 鹰眼 | ATK+30%/CRIT+20% | +2%/+1% | ATK+48%/CRIT+29% |
| 素问金针 | ATK×100% | +5% | ATK×145% |
| 回春术 | ATK×320% | +10% | ATK×410% |
| 益气汤 | ATK×180% | +7% | ATK×243% |
| 悬壶 | ATK×40% | +2% | ATK×58% |
| 百战之躯 | +3%/10%HP | +0.15%/10%HP | +4.35%/10%HP(上限43.5%) |
| 通灵 | 冷却 -1 | 不变 | 不变（机制不变） |
| 追猎 | 60% 追击 | +2% | 78% 追击 |

> 以上数值为暂定方向，**不代表最终定稿值**。精确曲线在"数值系统设计 - 属性成长曲线"阶段确定。
>
> 被动技能中通灵和追猎的机制不随等级变化，但追猎的追击伤害倍率随等级提升。

### 8.3 升级消耗

- **消耗资源**：游戏币 + 技能书（各职业专属掉落）
- **成本曲线**：`cost(SLv) = base × SLv × multiplier`（SLv 越高越贵）
- **总等级**：SLv 1~10，共需升级 9 次/角色

资源产出途径：副本掉落、商店购买、活动奖励。

### 8.4 为何用角色技能等级而非单个技能等级

- **减少管理负担**：3 角色 × 4 技能 = 12 个独立升级项，对微信小游戏玩家来说太多
- **决策聚焦**：升级角色 = 全技能提升，玩家的决策是"升哪个角色"而不是"升哪个技能"
- **弱化"只升一个技能"的极端玩法**：避免出现"全队只升破甲箭，其他都 1 级"的畸形配装
- **保持技能组合完整性**：每个角色的技能组合是设计好的，全升才能体验完整

## 九、自动战斗 AI 逻辑

### 9.1 总则

自动战斗模式下，AI 按优先级选择技能。普攻是兜底选项（无冷却，无消耗）。

**AI 决策流程（每个角色行动时）：**

```
1. 检查是否有可用（未冷却）的主动技能
   ├─ 有 → 按优先级列表检查条件，使用第一个条件满足的技能
   └─ 无 → 使用普攻

2. 目标选择（如果是单体技能）：
   按目标选择规则选择最优目标

3. 执行技能
```

### 9.2 玄甲 AI

| 优先级 | 条件 | 技能 | 目标选择 |
|:------:|------|------|---------|
| 1 | 铁壁嘲讽可用且当前未被嘲讽 | 铁壁嘲讽 | 全体（不需要选目标） |
| 2 | 护盾援护可用且有队友（非自身）HP < 50% | 护盾援护 | 当前 HP% 最低的队友 |
| 3 | 护盾援护可用且队友的护盾即将消失（剩余 0 轮） | 护盾援护 | 护盾即将消失的队友 |
| - | 以上都不满足 | 盾击（普攻） | 当前 HP% 最低的敌人（争取击杀） |

**设计意图：** 玄甲的 AI 保持"保护优先"的原则。嘲讽是第一优先（越早开嘲讽，队友越安全），护盾是第二优先（急救残血队友），普攻是保底。目标选择优先低血量敌人——虽然玄甲输出低，但补刀一个残血的价值大于打满血目标。

### 9.3 方士 AI

| 优先级 | 条件 | 技能 | 目标选择 |
|:------:|------|------|---------|
| 1 | 缚灵咒可用且有敌方单位 SPD > 120 | 缚灵咒 | 当前 SPD 最高的敌方 |
| 2 | 烈焰符可用且敌方存活数 ≥ 2 | 烈焰符 | 全体（AOE） |
| 3 | 烈焰符可用但敌方只剩 1 个 | 烈焰符 | 仍使用（单体 AOE ≈ 普攻伤害，但 AOE 本身有仇限风险） |
| 4 | 缚灵咒可用 | 缚灵咒 | 当前 HP% 最低的敌人（争取触发通灵） |
| - | 以上都不满足 | 符箭（普攻） | 当前 HP% 最低的敌人 |

**设计意图：** 方士 AI 优先减速高速敌人（控场价值），其次在有多个敌人时 AOE。当只有一个敌人时，缚灵咒的单体伤害（180%）高于烈焰符的 AOE（130%），所以缚灵咒在单目标场景优先级回升。通灵的击杀 CD 缩减是隐藏收益——AI 倾向打残血目标触发通灵。

### 9.4 惊鸿 AI

| 优先级 | 条件 | 技能 | 目标选择 |
|:------:|------|------|---------|
| 1 | 破甲箭可用且鹰眼增益已激活 | 破甲箭 | DEF 最低的敌方（最大化暴击收益） |
| 2 | 鹰眼可用且当前无鹰眼增益 | 鹰眼 | 自身 |
| 3 | 破甲箭可用且鹰眼增益未激活 | 破甲箭 | HP% 最低的敌方（争取击杀触发追猎） |
| - | 以上都不满足 | 速射（普攻） | HP% 最低的敌方 |

**设计意图：** 惊鸿 AI 有组合技意识——优先在鹰眼增益下使用破甲箭（最大化 combo 价值）。如果破甲箭在 CD 且鹰眼可用，先开鹰眼为下轮爆发做准备。目标选择逻辑：有鹰眼时选低防御（最大化伤害），无鹰眼时选低 HP（争取击杀触发追猎）。

### 9.5 素问 AI

| 优先级 | 条件 | 技能 | 目标选择 |
|:------:|------|------|---------|
| 1 | 回春术可用且有队友 HP ≤ 30% 或 | 回春术 | HP% 最低的队友 |
| | 队友 HP ≤ 50% 且身上有 debuff | | |
| 2 | 益气汤可用且有 ≥ 2 名队友 HP ≤ 50% | 益气汤 | 全体 |
| 3 | 回春术可用且有队友 HP ≤ 50% | 回春术 | HP% 最低的队友 |
| 4 | 益气汤可用且有 ≥ 2 名队友 HP ≤ 70%（预防性） | 益气汤 | 全体 |
| - | 以上都不满足 | 金针（普攻） | HP% 最低的队友 |

**设计意图：** 素问 AI 按血量阈值分级响应：
- **30% 以下**：红色警戒，直接回春术急救（最高优先级）
- **30%~50%**：如果有 debuff（怕 debuff 导致后续伤害加重），回春术优先用于净化；如果没有 debuff，群奶或回春术都可以
- **50%~70%**：预防性群奶（益气汤），适合应对即将来临的 AOE
- **70% 以上**：金针 + 悬壶的被动节奏就够了

回春术的优先级高于益气汤（单奶量大、冷却短、附带净化），只在多人残血时益气汤才优于回春术。

### 9.6 AI 行为边界情况

| 场景 | 处理 |
|------|------|
| 所有技能均在冷却 | 使用普攻 |
| 技能可用但条件不满足 | 跳过，检查下一优先级。如果所有技能条件都不满足 → 普攻 |
| 全员满血（素问） | 金针自动奶最低血（即使满血也会选择目标，但治疗量溢出） |
| 护盾目标已满血（玄甲） | 仍然套盾（护盾是预防性保护，和当前血量无关） |
| 所有目标都 100% HP | 金针选位置最靠前的队友（固定规则） |

## 十、技能交互规则

### 10.1 Buff 叠加规则

**定义：** Buff 是正面效果（来自己方技能），Debuff 是负面效果（来自敌方技能）。

**同类 Buff 不叠加：**
- 同一属性类别的 Buff 只取最高值生效
- 例如：玄甲的 DEF+20%（铁壁嘲讽）不与装备上的 DEF+10% 叠加为 +30%，而是取最高值 +20%
- 持续时间各自独立计算（但被覆盖的 Buff 虽然不生效，到期后下级 Buff 也不会"恢复"——已被覆盖的效果在覆盖时丢失）

**异类 Buff 共存：**
- 不同属性的 Buff（ATK+30% 和 DEF+20%）同时生效，互不影响
- 在 Step 0 计算面板属性时全部累加

**同名 Buff 刷新：**
- 如果同一技能再次施加同名 Buff，重置持续时间为完整时长
- 例如：铁壁嘲讽的 DEF+20% 在持续第 2 轮时再次使用，持续时间重置为 2 轮

**护盾（特殊）：**
- 护盾不是属性类 Buff，是独立的 HP 缓冲区
- 新护盾替换旧护盾（不叠加，取新值重置持续时间）

### 10.2 Debuff 交互规则

**同类 Debuff 不叠加：**
- 同一属性类别的 Debuff（如 SPD-20%）只取最高值生效
- 不同来源的同类 Debuff 只刷新持续时间（取最长剩余时间？还是重置为完整持续时间？）
  - **规则**：新 Debuff 施加时，如果目标已有同类 Debuff，刷新持续时间为新 Debuff 的时长

**异类 Debuff 共存：**
- 不同类型的 Debuff（SPD-20% 和 DEF-15%）同时生效

**Debuff 与 Buff 互斥？**
- 同一属性上的 Buff 和 Debuff 可以同时存在，但实际结算时取总和（正负抵消）
- 例如：ATK+30%（鹰眼）和 ATK-10%（假设某敌人技能），实际 ATK% 修正 = +20%
- 如果总和为负，降低该属性；总和为正，提升该属性

**净化的边界：**
- 净化只移除 `Debuff` 标记的效果
- `Buff` 标记的效果不受净化影响
- 不可净化的特殊 Debuff（如 Boss 专属标记）在具体技能中标注 `[不可净化]`

### 10.3 同时触发判定

当多个效果在同一个事件点触发时，按以下顺序处理：

**击杀事件触发顺序：**

```
1. 伤害结算 → 目标 HP ≤ 0 → 标记"即将阵亡"
2. 触发击杀方被动的"on kill"效果：
   a. 方士「通灵」：冷却 -1（多个击杀多次触发）
   b. 惊鸿「追猎」：触发追击（仅击杀者=惊鸿时，每轮最多一次）
3. 如果追击产生击杀：
   a. 方士「通灵」再次触发（因为击杀产生时方士可以看到）
   b. 惊鸿「追猎」不再触发（已达每轮上限）
4. 移除阵亡单位
```

**回合结束阶段触发顺序：**

```
1. 触发"每轮结束时"效果
2. 仇恨值衰减 ×80%
3. Buff/Debuff 计时 -1
4. 移除到期效果（计时 ≤ 0 的 Buff/Debuff）
5. 胜负判定
```

**素问行动结束时触发顺序：**

```
1. 素问完成主动/普攻技能
2. 素问触发「悬壶」：全队治疗
3. 悬壶产生的仇恨值实时计入
4. 素问的回合正式结束
```

**同时触发多个相同效果时：**
- 如果两个被动同时监听同一事件（比如两个玄甲的百战之躯），按队伍位置顺序处理
- 本游戏队伍中不可能出现两个同角色，但可能有两个同职业（如双玄甲队）
- 规则：按队伍位次（1→2→3）依次结算

### 10.4 技能优先级速查

| 结算层级 | 内容 | 备注 |
|---------|------|------|
| Layer 0 | 面板属性修正（Step 0） | Buff/Debuff 影响基础属性 |
| Layer 1 | 技能伤害/治疗计算 | Step 1~6 |
| Layer 2 | Buff/Debuff 最终修正（Step 7） | 增伤/减伤在此层 |
| Layer 3 | 保底（Step 8） | 至少 1 伤害 |
| Layer 4 | 击杀判定 | HP ≤ 0 时触发 |
| Layer 5 | On-kill 效果 | 通灵、追猎等 |
| Layer 6 | 仇恨实时产生 | 伤害/治疗/护盾/增益各算各的 |
| Layer 7 | 行动结束触发 | 悬壶等 |

## 十一、边界情况处理

### 11.1 技能冲突

| 场景 | 处理方式 |
|------|---------|
| 玄甲被控时能否开嘲讽 | **不能**——被控制状态无法使用主动技能（控制类型在 Buff/Debuff 阶段定义） |
| 素问对自己使用回春术 | **可以**——回春术目标可以是任何友方，包括施法者自己 |
| 护盾目标在护盾持续期间死亡 | 护盾随目标死亡消失（目标死亡后所有效果移除） |
| 护盾援护对满血目标使用 | **可以**——护盾是额外缓冲区，和当前血量无关。推荐在战斗中预判伤害提前套盾 |
| 惊鸿的追猎触发时全场只有 1 个敌人 | **不触发**——需要"另一个"存活目标。如果只剩 1 个敌人且被击杀，追猎没有可追击的目标 |
| 方士 AOE 击杀所有敌人 | 正常触发通灵（每个击杀 -1CD），战斗结束判定在击杀后执行 |
| 素问被控制时悬壶触发吗 | **不触发**——被控制时"行动未完成"，无法触发"行动结束时"的效果 |
| 素问行动结束时全队满血 | 悬壶仍触发，治疗量为 0（不产生仇恨） |

### 11.2 职业搭配的自动战斗预期

| 编队 | AI 行为预期 | 评价 |
|------|------------|------|
| 玄甲+方士+惊鸿（标准队） | 玄甲嘲讽保人，方士烈焰符压血线，惊鸿破甲箭点杀 | 配合完善，自动战斗效率高 |
| 玄甲+素问+惊鸿（无方士队） | 缺少 AOE 能力，惊鸿单体逐一击杀，节奏较慢 | 可用但打群怪吃力 |
| 玄甲+方士+素问（无惊鸿队） | 有 AOE 有治疗有坦克但缺单体爆发，Boss 战输出不足 | 苟命队，适合高难本 |
| 双玄甲+素问 | 双重嘲讽覆盖没有空窗期，但输出严重不足 | 极端苟命，不适合常规推图 |
| 三方士 | 三倍 AOE 快速清场但全员脆皮且无治疗 | 赌狗队，清杂兵快，遇 Boss 暴毙 |

## 十二、实现注意汇总

### 12.1 数据结构建议

```javascript
// 技能定义
class Skill {
  id: string;           // "xuanjia_shield_bash"
  name: string;         // "盾击"
  type: 'normal' | 'active' | 'passive';
  baseMultiplier: number;   // 基础倍率（Lv1）
  perLevelBonus: number;    // 每级倍率提升
  cooldown: number;         // 主动技能的冷却轮数
  hateRules: {
    type: 'damage' | 'heal' | 'shield' | 'buff' | 'fixed';
    multiplier: number;       // 对应×0.7/×0.35/×0.3/30/500 等
    range: 'target' | 'all_enemies';
  };
  conditions?: {            // 被动触发条件
    trigger: 'on_kill' | 'on_damage_taken' | 'on_action_end';
    limit?: number;           // 每轮上限
  };
}

// 战斗中的技能实例
class SkillInstance {
  skillId: string;
  currentCooldown: number;    // 当前剩余冷却（0 = 可用）
  level: number;              // 技能等级
  getMultiplier(): number;    // 基于等级计算当前倍率
}
```

### 12.2 性能关注点

- **AOE 伤害计算**：烈焰符最多 6 目标，每个目标独立走一遍伤害公式（约 8 步整数运算），单次技能约 48 步运算。微信小游戏 JS 环境轻松处理
- **被动触发频率**：通灵和追猎绑定击杀事件，一场战斗触发次数有限（<= 敌人数量）
- **仇恨矩阵大小**：`O(playercount × enemycount)`，3v6 场景最多 18 个整数，极小
- **护盾计算**：每次伤害多一步"护盾值扣减"判断，开销可忽略

- 主动技能按钮上显示冷却轮数（数字倒计时）

- 素问的自动奶应有清晰的目标选择表现（选中目标高亮）
