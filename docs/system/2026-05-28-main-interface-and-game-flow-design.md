日期：2026-05-28
状态：approved

## 一、设计原则

1. **字符优先**：全界面使用 Unicode 字符（框线、符号、进度条），无图片/图标
2. **移动端适配**：内容宽度以 ~40 英文字符（~20 汉字）为准，单手操作友好
3. **沉浸感**：中式古风命名，界面氛围统一，避免"系统设置"式的冷冰冰布局
4. **导航清晰**：任何时候按底部 Tab 可直达核心功能，不超过 3 次点击到达任何页面

### 核心循环

```
   ┌──────────────────────────────┐
   │                               │
   ▼                               │

│ 打关  │───→│ 拿资源 │───→│ 养成  │───┘
└──────┘    └──────┘    └──────┘
  (闯关)    (经验/金币   (升级角色
             /材料/装备)   /强化装备
                           /提升技能)
```

### 2.1 体力系统

体力是关卡挑战的消耗资源，支撑游戏核心循环。

| 项目 | 数值 | 说明 |
|------|------|------|
| 上限 | 120 点 | 满级后不再自然回复 |
| 回复速度 | 1 点 / 6 分钟 | 即 12 小时回满 |
| 关卡消耗 | 主线关卡 10 点/次 | 扫荡同消耗 |
| 回复道具 | 干粮（+60 点）| 升级/活动/商店获得 |
| 付费补充 | 玉璧购买 | 每日限购次数上限待定 |

> 体力是基础闭环：打关 → 消耗体力 → 等回复/吃道具 → 继续打关。
> 具体数值可在开发调优阶段调整，此处为占位设计使流程完整。

### 3.2 各区域说明

**顶栏**（固定高度 3 行）：
- 第 1 行：左侧标题"游侠阁"，右侧游侠等级（账号等级）+ 设置按钮
- 第 2 行：两种货币显示（金币 = 通用货币，玉璧 = 稀有货币/抽卡道具）
- 顶栏的 Lv.XX 代表**账号等级（游侠等级）**，角色等级在队伍/详情区域单独显示

**功能入口**（2×3 网格，每个按钮 10 宽 × 3 高）：
- 闯关：主线关卡入口，右上角可标红点（新关卡开放提示）
- 角色：角色列表与详情管理
- 招募：抽卡/招募入口，右上角可标红点（免费抽卡次数）
- 店铺：内购商城（皮肤/月卡/礼包）
- 每日：每日任务/活跃度
- 江湖：图鉴/剧情回顾/成就等收集类内容

**当前队伍**（快捷概览区域）：
- 显示编队中 3 个角色的职业、名字、等级、HP 条（大概位置）
- 点击任意角色 → 跳转到角色详情
- 设计意图：玩家回主界面一眼看到队伍状态，方便判断要不要提升

**最新动态**（信息流）：
- 战斗结果、新角色获得、成就解锁等消息
- 只显示最近 3 条，滚动更新

**底部导航**（固定 4 项）：
- 主页 → 主界面
- 编队 → 队伍编辑页
- 背包 → 物品/装备列表
- 招募 → 抽卡页

### 3.3 为什么这样布局

- 主页承担"据点/大厅"角色，不只是功能中转站，也有氛围感
- 6 个功能入口的网格是移动端经典布局，触控区域够大（10 字宽，手指不会点错）
- 当前队伍在主界面直接可见，养成和战斗之间的反馈链路短——练了角色回主页就能看到变化
- 底部导航 4 项是最优数量：太少要频繁跳转，太多在小屏上拥挤
- 角色详情从多个入口都能到（主页→角色、编队点击角色、背包→装备），不依赖唯一路径

## 四、关卡选择与关卡地图

### 4.1 章节→关卡 两级结构

```
┌─────────────────────────────────────────┐
│  ── 第 一 章 · 初 入 江 湖 ──           │
│                                         │
│  【已完成】 1-1  初出茅庐  ★★★         │
│  【已完成】 1-2  山贼拦路  ★★★         │
│  【已通关】 1-3  密林深处  ★★☆          │
│  【☆☆☆☆☆】 1-4  猛虎下山  ▶ 进行中    │
│  【🔒 未解锁】 1-5  山寨首脑  Lv.15     │
│                                         │
│  ★ 总进度: 3/5    扫荡: 1-1  1-2       │
│                                         │
│  [← 返回章节列表]                       │
└─────────────────────────────────────────┘
```

### 4.2 关卡卡片内容

每个关卡显示：
- 关卡编号 + 名称
- 星级评价（★ 满 3 星）
- 推荐等级（未解锁时显示）
- 状态标记：【已完成】【已通关】【进行中】【未解锁】

第一次通关 = "通关"，获得星级评价 + 首通奖励
之后重复挑战 = "扫荡"，直接按星级结算，不进战斗

### 4.3 章节列表（从主界面→闯关 进入）

```
┌─────────────────────────────────────────┐
│  第 一 章 · 初入江湖   第 1-5 关         │
│  推荐等级 Lv.1-15   进度 ★★★★☆         │
│  ───────────────────────────────────    │
│  第 二 章 · 幽暗密林  未开启             │
│  推荐等级 Lv.15-25  条件: 通关第一章     │
│  ───────────────────────────────────    │
│  第 三 章 · 碧水寒潭  未开启             │
│  推荐等级 Lv.25-35  条件: 通关第二章     │
│  ───────────────────────────────────    │
│  ...                                     │
└─────────────────────────────────────────┘
```

### 4.4 星级评价标准

| 条件 | ★ 数 |
|------|------|
| 战斗胜利 | ★（基础） |
| 无人阵亡 | ★ |
| 10 轮内结束 | ★ |

满星 3 星。扫荡时按历史最高星级结算奖励。

> 符号说明：★ 在不同场景含义不同——
> - **关卡评价**：★ 为战斗评价星级（满 3 星），见上表
> - **角色资质**：★ 为角色稀有度星级（满 5 星），如 ★★★★☆ 表示四星角色
> - **章节进度**：★ 为章节收集进度（满 N 星 = 关卡数 × 3），如 ★★★★★★☆☆☆☆ 表示章节总进度

### 战斗画面说明

- 分四块区域：敌方面板、我方面板、行动日志、操作栏
- 当前行动单位高亮（如果是敌方行动，操作栏隐藏或灰显）
- 行动日志显示最近 2-3 条行动记录
- 操作栏根据当前行动单位显示可用技能（最多 5 个技能，不够用 "..." 收折）
- 道具按钮 → 弹出道具选择列表
- 自动按钮 → 切换自动战斗

### 敌方显示简化规则

- 敌方 > 3 个时，低威胁小怪折叠显示（显示剩余数量 + 总 HP 条）
- 头目/Boss 始终单独显示
- 具体折叠策略取决于关卡设计，框架预留即可

### 8.4 红点系统规则

红点用于引导玩家关注可操作内容，不承载数值信息（不显示数字）。

| 触发位置 | 触发条件 | 清除条件 |
|---------|---------|---------|
| 主页「闯关」入口 | 新关卡解锁 | 点击进入关卡列表后清除 |
| 主页「每日」入口 | 有未领取的每日/周常奖励 | 领取奖励后清除 |
| 主页「招募」入口 | 免费招募次数可用（每日 1 次免费单抽） | 使用免费招募后清除 |
| 主页「江湖」入口 | 有新解锁的图鉴/剧情 | 查看对应条目后清除 |
| 底部「背包」Tab | 获得新物品且未查看 | 打开背包浏览后清除 |
| 底部「招募」Tab | 同主页招募入口红点 | 同步清除 |
| 角色详情「技能升级」| 有可升级的技能 | 升级或查看后清除 |
| 角色详情「装备」| 有可更换的更强装备 | 进入装备界面后清除 |

**规则要点**：
- 所有红点同步实时刷新，不依赖手动刷新
- 红点只标记"新内容可查看"，不标记"有内容未读"（避免泛滥）
- 同源红点（如主页和底部的招募）共用触发条件

### 8.5 操作反馈规范

字符界面下无动效/音效，操作反馈依赖文本即时变化。

| 操作 | 反馈方式 | 示例 |
|------|---------|------|
| 点击按钮 | 按钮文本闪烁一次（由 `[按钮]` → `【按钮】` → `[按钮]`） | 点击 `[开始战斗]`，短暂变为 `【开始战斗】` 后恢复 |
| 点击可点击行 | 行首 `▶` 变为 `▷` 后恢复，或整行反色一瞬间 | |
| 切换 Tab | 选中 Tab 标记切换（`◈ 主页` → `◆ 主页` 或加下划线） | |
| 返回操作 | 画面顶部弹出 `← 返回中...` 占位文字，然后跳转 | |
| 加载中 | 显示加载动画（三个 `◇` 依次亮起） | ◇ ◇ ◇ → ◆ ◇ ◇ → ◇ ◆ ◇ → ◇ ◇ ◆ → 循环 |
| 确认/提交 | 按钮变为 `【确认中...】`，完成后恢复正常 | |
| 错误/失败 | 按钮闪烁红色标记或在操作栏显示一行红色提示 | `!! 体力不足，无法挑战` |
| 禁用状态 | 按钮变为 `(灰显文字)` 圆括号包围，不可点击 | `(开始战斗)` 表示条件未满足 |

**设计原则**：
- 每次点击必须有即时视觉反馈（哪怕只是符号变化），不能死板
- 反馈时间 < 300ms（字符变化足够快）
- 按钮状态三态：可用 `[文字]` → 点击 `【文字】` → 禁用 `(文字)`
- 无法满足条件时明确告知原因（如"体力不足""等级不足"），而非单纯禁用

## 九、设计要点总结

1. **导航结构 4+1**：4 个底部 Tab 保底 + 主页作为功能集散地，任何页面最多 3 次点击可达
2. **主页即大厅**：主页不只是功能列表，显示当前队伍概览 + 最新动态，让玩家感觉"回到据点"
3. **关卡两级结构**：章节 → 关卡，线性解锁，标准手游关卡模式，实现成本最低
4. **战斗屏幕四分区**：敌方面板 / 我方面板 / 行动日志 / 操作栏，信息层级从上到下：敌人→自己→发生了什么→我要做什么
5. **角色详情是核心**：属性、技能、装备、故事全在一个页面上，多条路径都能到达
6. **字符界面规范统一**：框线、标记、进度条风格一致，让字符风格成为游戏特色而不是"简陋"
7. **空状态有引导**：空列表不干巴巴显示"暂无"，而是给引导操作入口
